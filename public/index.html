AstarothCipher Protocol

Local-first, zero-knowledge cryptographic file exchange.

1. Establishing Identity

Before you can receive encrypted files, you must establish a cryptographic identity. Navigate to the Key Manager to generate your RSA-2048 keypair entirely in your browser's RAM.

  • Public Key: Your public address. Share this .pem file safely with anyone who wants to send you a secure file.
  • Private Key: Your master vault key. It is used to decrypt files sent to you. Never share this. Save it securely on your device.
Integrity Validation
During decryption, the engine verifies the AES-GCM authentication tag. If a single byte was tampered with in transit, decryption will automatically fail.

2. Encrypting (Sending)

To send a file to a recipient, you must possess their Public Key.

  • Go to the Encrypt tab.
  • Load their Public Key (paste it or upload the file).
  • Drag and drop the file you wish to send.
  • Click Encrypt & Download.

The engine generates a randomized AES session key, encrypts the file payload, and encrypts the AES key using the Public Key into an .hcry bundle.

3. Decrypting (Receiving)

When someone sends you an .hcry bundle, you need your Private Key to open it.

  • Go to the Decrypt tab.
  • Provide your Private Key.
  • Select the .hcry file you received.
  • Click Decrypt File to recover the original data.

Secure a File

Encrypt a file locally before sending it. You need the recipient's Public Key.

Recover Data

Decrypt an .hcry file sent to you. You must use your secret Private Key.

Identity Management

Generate your local cryptographic identity. Keys never leave your browser.
